# 分布式图查询系统 — test1（单机版）

## 概述

单文件实现。将分布式图查询的**核心算法**（一致性哈希分区、邻居查询、三角计数）以单机多进程形式跑在一个 Python 文件里，作为课程原始交付版本。

> 原始版本来自课程大作业，后续 test2-test4 基于此演进。

## 结构

单一文件 `distributed_graph_query.py`，约 550 行。

```
test1/
├── distributed_graph_query.py    ← 完整实现（单文件）
├── 实验报告_分布式图查询程序设计.md
└── 实验报告_分布式图查询程序设计.docx
```

## 功能

- **图数据生成** — Erdős–Rényi 随机图，支持 `G(n, p)` 模型
- **一致性哈希分区** — MD5 取模，将节点/边分配到 N 个分区
- **Coordinator** — 维护分区位置表，路由查询
- **Worker（子进程）** — `multiprocessing.Process` 模拟，在各自内存中加载分区数据
- **邻居查询** — 单节点邻居列表
- **共同邻居查询** — 两个节点的邻居交集
- **三角检测** — 逐边查询：对每条边 `(u,v)` 分别向两端 Worker 索取邻居，本地求交集
- **批量查询** — 支持同时查询多个节点的邻居列表
- **性能基准测试** — 内置 benchmark，测试 1k-10k 节点规模

## 运行

```bash
cd test1/
python3 distributed_graph_query.py
```

## 设计缺陷

这是初版实现，存在以下问题（已在 test2/test3 改进）：

1. **逐边 RPC** — 全图三角计数时，每条边都需要向两个 Worker 发起 RPC，O(E) 次通信
2. **长连接阻塞** — 使用 `subprocess` 管道通信，无并发能力
3. **无分布式部署** — 仅用 `multiprocessing` 模拟多主机，实际不可跨机器
4. **无容错** — Worker 崩溃即丢失数据
5. **接口简单** — 仅 Python 函数调用，无 TCP 网络 API
